# Services and Commercial Support

AIC is open and commercially usable under Apache-2.0.

That means teams can adopt it freely, including in commercial products and internal platforms. The business opportunity for AIC is not selling permission to use the code. It is selling expertise, support, implementation speed, and eventually hosted or premium capabilities around the ecosystem.

## Immediate Monetization Paths

- paid onboarding for teams making apps AIC-ready
- implementation services for React, Next.js, and Vite apps
- agent-readiness audits for risky workflows and production UI flows
- custom MCP integrations for Claude, Cursor, Codex, and internal agent stacks
- private support retainers and faster response channels
- training, architecture reviews, and integration workshops

## Later Monetization Paths

- hosted AIC dashboard and manifest review workflows
- managed bootstrap and review pipelines
- team collaboration, governance, and observability features
- enterprise support, SLAs, and compliance packaging
- premium integrations or managed cloud features

## Attribution Request

Apache-2.0 does not let this repository force product-level branding or public acknowledgment from every user.

What the project can do:

- preserve attribution through the repository and NOTICE file
- keep the AIC name and branding separate through trademark and naming policy
- ask adopters to mention AIC voluntarily

If you use AIC in a public product, please mention that your app is "built with AIC" and link back to the repository when practical:

- Repository: https://github.com/VPAI-Grok/AIC
- npm scope: `@aicorg/*`

## Positioning

The recommended strategy for AIC at this stage is:

- maximize adoption with Apache-2.0
- monetize implementation, support, and hosted value
- build the brand so the official AIC ecosystem remains the most trusted distribution
